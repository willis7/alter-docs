---
title: "APIゲートウェイがモデルをリストしない"
description: "外部アプリを Alter の API に接続する際の空のモデル リストを修正"
---

<Note>
  **API ゲートウェイを使用しますか?** これにより、Alter のモデルを SillyTavern、NovelCrafter などの他のアプリ、または独自のコードで使用できるようになります。
</Note>

## 何が起こっていますか？

別のアプリを Alter の API に接続しようとしていますが、次のようになります。
- アプリは接続しますが、利用可能なモデルが表示されません
- モデル名を手動で入力する必要があります
- 正確な名前を指定しないとモデルリクエストは失敗します

## なぜこれが起こるのか

多くの AI アプリは OpenAI の単純なモデル形式 (`gpt-5` など) を想定していますが、Alter は複数のプロバイダーをサポートしています。そのため、特別な形式を使用して、各モデルがどのプロバイダーから提供されているかを識別します。

## 解決策

<Card title="モデルの形式: プロバイダー#モデル名" icon="hashtag">
  `gpt-5` の代わりに、`OpenAI#gpt-5` を使用します。
  
  形式は常に次のとおりです: **プロバイダ名** + **#** + **モデル名**
</Card>

## セットアップ手順

<Steps>
  <Step title="API 認証情報を取得する">
    1. Alter → 設定 (`Cmd + ,`) を開きます
    2. **ルーター** タブに移動します
    3. [**新しいキーの追加**] をクリックして API キーを生成します
    4. キーをコピーします (`sk-...` で始まる)
    5. エンドポイント: `https://alterhq.com/api` に注目してください。
  </Step>
  
  <Step title="外部アプリを構成する">
    接続しているアプリ内:
    - **ベース URL:** `https://alterhq.com/api` (または `https://alterhq.com/api/v1` を試してください)
    - **API キー:** ステップ 1 で生成されたキー
  </Step>
  
  <Step title="モデルを手動で設定する">
    自動リストが機能しない可能性があるため、モデルを手動で入力します。
    
    |プロバイダー |モデル名 |完全なID |
    |----------|-----------|----------|
    | OpenAI | GPT-5 | __コード_0__ |
    | OpenAI | GPT-5 ナノ | __コード_0__ |
    | Claude |ソネット 4 | __コード_0__ |
    | Gemini | 2.5プロ | __コード_0__ |
    |ミストラル |大 | __コード_0__ |
    | Alter |ベストルーター | __コード_0__ |
  </Step>
</Steps>

## 接続をテストする

この簡単なテストを試してください。
1. モデル `OpenAI#gpt-5-nano` を使用してチャット完了リクエストを送信します
2. **成功:** Alter を通じて応答が返されるはずです。

1 つのモデルで動作する場合は、モデル ID を変更するだけで、すべてのモデルで動作します。

<Tip>
  **プロのヒント:** Alter API は OpenAI の API とまったく同じように機能します。アプリが「OpenAI 互換」エンドポイントをサポートしている場合、Alter で動作します。
</Tip>

## まだ問題がありますか?

- 有効な Alter API キー (プロバイダーのキーではない) を使用していることを確認してください。
- 両方のエンドポイントを試してください: `/api` と `/api/v1`
- 一部のアプリはモデル リストをキャッシュします。セットアップ後にアプリを再起動してみてください。
