## 利用制限

フェアユースの範囲では:
- 1日上限：`200`リクエスト/日
- 超過後: `10分あたり1リクエスト` に滞る
- 継続的に高い利用量が必要な場合: 予算をトップアップ

<Note>
  API Router は個人利用と低ボリュームのワークロード向けです。ユーザー製品や高ループスットのシステムには、プロバイダーの専用インフラを利用してください。
</Note>

## よくある問題

### モデルが表示されない
- モデルを `Provider#Model-name` 形式で手動指定する
- APIキーが有効か確認する
- クライアントが一覧モデルエンドポイントをサポートしているか確認する

### 認証エラー
- `Authorization: Bearer YOUR_API_KEY`を確認する
- エンドポイントが `https://alterhq.com/api` または `/api/v1` であることを確認する
- アカウントが有効であることを確認する

### 接続問題
- OpenAI互換クライアントを使用する
- `alterhq.com` へのネットワーク接続を確認する

### ハイレイテンシ
- 軽量モデルを挑戦（`OpenAI#gpt-5-nano`、`Alter#light`）
- 長文出力にはストリーミングを使う

## ベストプラクティス

1. APIキーは環境変数に保存する（ハードコードしない）
2. リクエスト量とエラー率を監視する
3.タスクの複雑さに合わせてモデル能力を選ぶ
4.バックオフ付きのリトライを実現する
5. ルーチン作業は低コストモデルを使う
