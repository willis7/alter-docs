## 挑戦

素晴らしいAlterセットアップが完了しました。
- チームのワークフローのためのカスタムアクション
- 意図的に設定された設定
- 重要なファイルを含む共有ワークスペース
- タスクに最適な特定のモデル

しかし、新しく誰かが参加するたびに、すべてを一から考えなければなりません。彼らは：
- 間違ったモデルを使用する
- カスタムアクションについてわからない
- すでに行った作業を再作成する
- 一貫性のない出力を行う

## 解決策: セットアップを共有する

Alterを使うと、構成を簡単に共有できるため、新しいチームメンバーは最初から生産性を高めることができます。

## 段階的なチームのオンボーディング

<Steps>
  <Step title="ベースライン構成を作成する">
    チームの標準設定を文書化します。
    - **推奨モデル** (例: 「コードレビューにはClaudeを使用し、クリエイティブなタスクにはGeminiを使用する」)
    - **無事の温度** (例:バランスの取れた創造性の場合は 0.7)
    - **音声ホットキー** (`Fn` またはカスタムキーで標準化)
    - **ワークスペースの命名規則**
  </Step>
  
<Step title="スターター Workspaceを構築する">
    共通のコンテキストを使用して共有ワークスペースを作成します。
    - **コードベース ワークスペース:** メイン プロジェクトを設定を含めます
    - **ドキュメント ワークスペース:** 仕様、要件、スタイル ガイドを含めます
    - **ナレッジ ベース ワークスペース:** 過去の決定事項、FAQ、Wiki を含めます
    
<Tip>
      `.alterworkspace` ファイルを直接共有することも、その作成方法を文書化することもできます。
    </Tip>
  </Step>
  
<Step title="カスタムアクションを共有する">
    チームの最も賢明なアクションを暴露します。
    1. **アクションエディタ**を開きます (メニューバーから)
    2. アクションを選択してください
    3. [**エクスポート**] をクリックします (または `Cmd + E` を押します)
    4. `.json` ファイルを保存します
    5. チームと共有する
    
新しいメンバーは、アクションエディターで `Cmd + O` を使用してこれらをインポートできます。
  </Step>
  
<Step title="チームプロファイルテンプレートを作成する">
    必須説明テンプレートを共有し、新しいメンバーが [設定] > [必須] に貼り付けることができます。
    
```
    Role: [Job title]
    Team: [Team name]
    Communication style: [Direct/casual/formal]
    
    Common tools we use:
    - GitHub repo: [URL]
    - Project management: [Linear/Asana/Jira]
    - Documentation: [Notion/Confluence]
    
    Technical context:
    - Primary language: [Python/JavaScript/etc]
    - Framework: [React/Django/etc]
    - Key dependencies: [List main libraries]
    ```
  </Step>
</Steps>

## 例: 開発チームがこれを使用する方法

**セットアップ:**
- **標準モデル:** クロードほとんどのタスク用のソネット
- **コード レビュー アクション:** セキュリティの問題、テスト、ドキュメントをチェックするために事前構成あり
- **共有Workspace:** メインのコードベース + APIドキュメント + 導入ガイド
- **チームプロフィール:** 全員に GitHub リポジトリ構造と主要な依存関係が含まれています

** 結果：**
新しい開発者が参加すると:
1. 変更をインストールします
2. チームのアクションをインポートする (5分)
3. 共有Workspaceのクローンを作成します (2分)  
4. 適切なテンプレートを貼り付けます (1分)
5. 彼らはすぐに生産性を発揮し、「認証システムはどのように機能するのか?」と問うことができます。状況に応じた答えを得る
