</Info>

## 挑戦

素晴らしい Alter セットアップが完了しました。
- チームのワークフローのためのカスタムアクション
- 慎重に設定された設定
- 重要なファイルを含む共有Workspace
- タスクに最適な特定のモデル

しかし、新しく誰かが参加するたびに、すべてを一から考えなければなりません。彼らは：
- 間違ったモデルを使用する
- カスタムアクションについてわからない
- すでに行った作業を再作成する
- 一貫性のない出力を行う

## 解決策: セットアップを共有する

Alter を使用すると、構成を簡単に共有できるため、新しいチーム メンバーは初日から生産性を高めることができます。

## 段階的なチームのオンボーディング

<Steps>
  <Step title="ベースライン構成を作成する">
    チームの標準設定を文書化します。
    - **推奨モデル** (例: 「コードレビューには Claude を使用し、クリエイティブなタスクには Gemini を使用する」)
    - **デフォルトの温度** (例: バランスの取れた創造性の場合は 0.7)
    - **音声ホットキー** (`Fn` またはカスタム キーで標準化)
    - **Workspaceのファイル命名規則**
  </Step>
  
  <Step title="スターター Workspaceを構築する">
    共通のコンテキストを使用して共有Workspaceを作成します。
    - **コードベース Workspace:** メイン プロジェクト フォルダーを含めます
    - **ドキュメント Workspace:** 仕様、要件、スタイル ガイドを含めます
    - **ナレッジ ベース Workspace:** 過去の決定事項、FAQ、Wiki を含めます
    
    <Tip>
      Workspaces は `~/Library/Application Support/Alter/Workspaces/` に保存されます。 `.alterworkspace` ファイルを直接共有することも、その作成方法を文書化することもできます。
    </Tip>
  </Step>
  
  <Step title="カスタムアクションを共有する">
    チームの最も役立つアクションをエクスポートします。
    1. **アクション エディタ**を開きます (メニュー バーから)
    2. アクションを選択してください
    3. [**エクスポート**] をクリックします (または `Cmd + E` を押します)
    4. `.json` ファイルを保存します
    5. チームと共有する
    
    新しいメンバーは、アクション エディターで `Cmd + O` を使用してこれらをインポートできます。
  </Step>
  
  <Step title="チームプロファイルテンプレートを作成する">
    プロファイル説明テンプレートを共有し、新しいメンバーが [設定] > [プロファイル] に貼り付けることができます。
    
    ```
    Role: [Job title]
    Team: [Team name]
    Communication style: [Direct/casual/formal]
    
    Common tools we use:
    - GitHub repo: [URL]
    - Project management: [Linear/Asana/Jira]
    - Documentation: [Notion/Confluence]
    
    Technical context:
    - Primary language: [Python/JavaScript/etc]
    - Framework: [React/Django/etc]
    - Key dependencies: [List main libraries]
    ```
  </Step>
</Steps>

## 例: 開発チームがこれを使用する方法

**セットアップ:**
- **標準モデル:** Claude ほとんどのタスク用のソネット
- **コード レビュー アクション:** セキュリティの問題、テスト、ドキュメントをチェックするために事前構成されています
- **共有Workspace:** メインのコードベース + API ドキュメント + 導入ガイド
- **チーム プロフィール:** 全員に GitHub リポジトリ構造と主要な依存関係が含まれています

**結果：**
新しい開発者が参加すると:
1. Alter をインストールします
2. チームのアクションをインポートする (5 分)
3. 共有Workspaceのクローンを作成します (2 分)  
4. プロファイル テンプレートを貼り付けます (1 分)
5. 彼らはすぐに生産性を発揮し、「認証システムはどのように機能するのか?」と尋ねることができます。状況に応じた答えを得る

## 定着させる

<AccordionGroup>
  <Accordion icon="refresh" title="毎週の改善セッション">
    毎週 15 分をかけて以下を確認してください。
    - 人々はどのアクションを使用していますか?
    - どのようなプロンプトが最も効果的ですか?
    - 新しいメンバーを混乱させるものは何ですか?
    
    フィードバックに基づいて共有リソースを更新します。
  </Accordion>
  
  <Accordion icon="users" title="チームチャットチャンネルを作成する">
    以下の専用チャンネル (Slack/Discord) を用意します。
    - 役立つプロンプトを共有する
    - 新しいアクションを投稿する
    - 「どうやって…？」質問
    - 勝利を祝う (「Alter のおかげで 2 時間節約できました!」)
  </Accordion>
  
  <Accordion icon="book" title="チームの戦略を維持する">
    Alter 設定をチーム Wiki に文書化します。
    - エクスポートされたアクションへのリンク
    - Workspaceのセットアップ手順
    - ドメインのベスト プラクティス
    - 避けるべきよくある間違い
  </Accordion>
</AccordionGroup>
