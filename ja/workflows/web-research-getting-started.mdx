このガイドで、Alter における最適な Web 検索構成を選びます。

<Info>
  次にAlterルーターを使ってください。`Fast`、`Best`、`Fair`、`Light`はWeb検索を標準搭載しています。
</Info>

## 使える選択肢

- **ローカル ツール:** `Web Search` と `Web Search Gemini`
- **リモート MCP サーバー:** Tavily、LinkUp、Exa
- **アプリギャラリー:** Firecrawl、LinkUp、Exa（ワークスペースで利用可能な場合）
- **Alterルーター:** `Fast`、`Best`、`Fair`、`Light` のネイティブ Web 検索

## プロバイダーの位置付け

- **Tavily:** AIエージェント向けのWebアクセス層。検索、抽出、調査、クローリングを単一APIで提供。
- **Firecrawl:** Web サイトを LLM 対応データへ変換。スクレイピングとクロールを重視します。
- **LinkUp:** AI アプリ向け検索。standard/deep モードとソース付き回答を訴えます。
- **Exa:** 品質重視のAI検索API。検索からリサーチまで複数のエンドポイントを提供。

## ハイブリッド戦略

1.まずはルーターのネイティブ検索から始めると、設定が最低限でデフォルトの結果も良好です。
2. Web 検索を持たないモデル（ローカルモデルを含む）には、Alter 同梱ツールで拡張します。外部アカウントは不要です。
3. どのモデルでも高い結果品質や詳細な制御が必要な場合は、MCP プロバイダーを追加します。
4. 複数ページの構造化抽出が必要な場合はスクレーパーを追加します。

<Warning>
  無料枠や上限は変更される場合がございます。ご利用前に最新情報をご確認ください。
</Warning>

<Tip>
  Web 検索ツールを同時に複数有効化しないでください。モデルのツールの選択ができなくなります。
</Tip>
