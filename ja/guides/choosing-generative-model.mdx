デフォルトでは、Alter はホスティング会社のアルファベット順にリストされたさまざまなモデルを提供します。

[APIキー](#via-your-api-key) または [あなたのローカルモデル](#locally-via-ollama-or-studiolm) を使用することもできます。

### モデルの選択 

Alter を開いたら、`/` と入力して利用可能なモデルを表示し、使用するモデルを選択します。


#### 好きなモデルを選ぶ

モデルの横にある星アイコンをマウスで選択し、そのモデルをお気に入りとしてマークします。
![モデルをお気に入りに登録する](/images/models/favorite-model.webp)


API キーまたはローカル モデルを使用する場合、これらは上部の **カスタム** の下に表示されます。  
他のすべてのインタラクションは Alter クラウド経由でルーティングされます。

### Alter クラウドの使用

デフォルトでは、 Alter は内部モデルルーターである `/best` を使用してセットアップされ、クエリに基づいて最適なモデルを選択します。

![ルーターとモデルの例](/images/settings/router.png)


**[設定] > [デフォルト] > [モデル]** でデフォルトのモデルを変更できます。  


#### Alter クラウド経由のモデルのリスト

Alter は、統合ルーターを通じて **10 プロバイダーにわたる 92 以上のモデル**へのアクセスを提供します。 `/models` エンドポイントを使用して現在のモデルを一覧表示するか、**[API ゲートウェイ ガイド](/guides/api-gateway)** でモデルの詳細情報を参照してください。

**92 を超えるすべてのモデルの現在の完全なリストについては、API を介して直接利用可能なモデルを確認してください。

```bash
curl https://alterhq.com/api/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

または、モデルの選択と使用方法の詳細については、**[API ゲートウェイ ガイド](/guides/api-gateway)** を参照してください。

### API キーの使用

![カスタムローカル](/images/models/custom-api-keys.webp)

独自の API キーを使用して、プロバイダーのエンドポイントに直接接続できます。  
1. **[設定] > [API キー] > [カスタム プロバイダー]** に移動します。  
2. カスタムプロバイダーを有効にする
3. リストからプロバイダーを選択してください

> 接続すると、**カスタム エンドポイント**の横にチェックマークが表示されます。  

プロバイダーのモデルは [**カスタム**] セクションにリストされます。

> 私たちは OpenAI 互換エンドポイントのみをサポートしています。つまり、Anthropic API キーは使用できません。

提供されたイメージに基づいて、カスタム キーがサポートされているプラ​​ットフォームのリストを次に示します。

1.  アズール
2.  Gemini
3.  LMスタジオ
4.  ミストラル
5.  オラマ
6.  OpenAI
7.  オープンルーター
8.  カスタム (任意の OpenAI 互換エンドポイント)

#### Alter での Azure エンドポイントのセットアップ

1.  **カスタム プロバイダーを有効にする:** Alter の設定で、「カスタム プロバイダー」オプションを有効にします。
2.  **Azure を選択します:** プロバイダーとして Azure を選択します。
3.  **完全なチャット完了エンドポイント URL を入力してください:** `api-version` パラメーターと URL 内のモデル名を含む、完全なチャット完了エンドポイント URL を入力します。  例: `https://yourinstance.openai.azure.com/openai/deployments/your-model-name/chat/completions?api-version=2024-02-15-preview`
4.  **API キーを入力してください:** Azure OpenAI API キーを入力します。

重要な考慮事項

*   **完全なエンドポイントが必要です:** Alter は URL からモデル名を抽出するため、*完全な* チャット完了エンドポイントを必ず使用してください。
*   **URL のモデル名:** モデルのデプロイメント名は URL パスに *必ず*含まれます。
*   **API バージョン:** URL には `api-version` パラメーターが *必ず*含まれます。

### Ollama または LM Studio をローカルで使用する

![カスタムローカル](https://beehiiv-images-production.s3.amazonaws.com/uploads/asset/file/a3cb944c-7379-49a6-9349-c829bcda071b/image.png?t=1740499713)

ローカル モデルで Alter を使用するには (Ollama または LM Studio 経由):  
1. **[設定] > [API キー] > [カスタム プロバイダー]** に移動します。  
2. カスタムプロバイダーを有効にする
3. プロバイダーのリストから Ollama または LM Studio を選択します

> 接続すると、**カスタム エンドポイント**の横にチェックマークが表示されます。  

![画像削除](https://beehiiv-images-production.s3.amazonaws.com/uploads/asset/file/4a22f3fb-ad9e-4f50-8184-1e5cc5238118/Alter_Settings_API_key_custom_provider_on.png?t=1740500360)

ローカル モデルは [**カスタム**] セクションにリストされます。

![カスタムモデル](https://beehiiv-images-production.s3.amazonaws.com/uploads/asset/file/c3a9bef5-ff23-4a35-99b6-aedc7bc38972/Alter_Main_Custom_Models.png?t=1740502675)
