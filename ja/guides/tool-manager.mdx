## ゴール

明確なトラブルシューティング手順により、統合を安全に接続および管理します。

## ステップ

1. Alter 設定を開き、統合またはTools Managerに移動します。
2. 一度に 1 つの統合を接続し、すぐにテストします。
3. 必要な権限とアカウントのスコープを確認します。
4. 簡単なテスト プロンプトを作成して、予想される動作を確認します。
5. 失敗した場合は、プロバイダー認証を確認して再試行してください。

## 多数の Tools の管理

多くのツールを有効にすると、コンテキスト ウィンドウの使用量が増加し、パフォーマンスに影響します。ほとんどのワークフローでは、リクエストごとに適切なツールを自動的に選択する **Flow** - Alter のツール オーケストレーターを使用します。

<Card title="Tool オーケストレーションには Flow を使用する" icon="wand-magic-sparkles" href="/guides/use-flow-orchestration">
  Flow は、リクエストを適切な統合に自動的にルーティングする単一のツールとして機能することで、コンテキストの使用量を削減します。
</Card>

すべてのツールをすべてのリクエストにロードするのではなく、Flow:
- リクエストを分析し、適切なツールを選択します
- 複数のツールにわたるマルチステップのオーケストレーションを処理します
- コンテキスト ウィンドウの使用量を減らし、パフォーマンスを向上させます。

**Local Tools > Alter** の Tool マネージャーで Flow を有効にします。

## トラブルシューティング

- プロバイダ接続を再認証します。
- ローカル/ネットワークの権限を確認します。
- 統合固有の制限とクォータを確認します。
