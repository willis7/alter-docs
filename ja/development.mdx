<Info>
  **前提条件**:
  - Node.js バージョン 19 以降
  - Git
</Info>

次の手順に従って、Alter ドキュメントをローカルでプレビューし、変更を提供します。

<Steps>
<Step title="リポジトリのクローンを作成する">

```bash
git clone https://github.com/wearedevx/alter-docs.git
cd alter-docs/docs
```
</Step>

<Step title="Mintlify CLIをインストールする">

```bash
npm i -g mint
```
</Step>

<Step title="ローカルでプレビューする">

docs ディレクトリで次のコマンドを実行します。

```bash
mint dev
```

ドキュメントのローカル プレビューは `http://localhost:3000` で入手できます。

</Step>
</Steps>

## カスタムポート

デフォルトでは、Mintlify はポート 3000 を使用します。 `--port` フラグを使用してポートをカスタマイズできます。

```bash
mint dev --port 3333
```

## リンクの検証

変更を送信する前に、壊れたリンクがないか確認してください。

```bash
mint broken-links
```

## AI 支援ライティング

Mintlify と連携するように AI コーディング ツールをセットアップします。

```bash
npx skills add https://mintlify.com/docs
```

これにより、Claude コード、カーソル、ウィンドサーフィンなどの AI ツール用の Mintlify のドキュメント スキルがインストールされます。

## 導入

変更は、GitHub 統合を通じてメイン ブランチにマージされると自動的にデプロイされます。

## 貢献ガイドライン

貢献する前に、以下をお読みください。
- [ALTER.md](/ALTER) - AI エージェントの貢献ルール
- [エージェント.md](/AGENTS) - エージェント向けのプロジェクト手順

重要なポイント:
- YAML フロントマターで MDX 形式を使用する
- 既存のコンテンツ構造に従う
- アセットにローカル `/images/` パスを使用する
- 送信する前に `mint broken-links` を実行してください

## トラブルシューティング

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    これはノードのバージョンが古いことが原因である可能性があります。次のことを試してください。
    1. 現在インストールされている CLI のバージョンを削除します: `npm remove -g mint`
    2. Node v19 以降にアップグレードします。
    3. CLI を再インストールします: `npm i -g mint`
  </Accordion>

  <Accordion title="問題: 不明なエラーが発生しました">
  
    解決策: デバイスのルートに移動し、`~/.mintlify` フォルダーを削除します。次に、`mint dev` を再度実行します。
  </Accordion>
</AccordionGroup>

最新の CLI バージョンで何が変更されたか知りたいですか? [CLI変更ログ](https://www.npmjs.com/package/mintlify?activeTab=versions)を確認してください。
