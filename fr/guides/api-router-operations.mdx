## Limites d'utilisation

Sous fair use :
- Limite quotidienne : `200` requêtes/jour
- Après la limite : limité à `1 requête / 10 minutes`
- Pour un volume régulier plus élevé : rechargez votre budget

<Note>
  API Router est conçu pour un usage personnel et des charges faibles. Pour des produits multi-utilisateurs ou un trafic agentic élevé, utilisez une infrastructure dédiée chez les providers.
</Note>

## Problèmes courants

### Modèles non listés
- Définissez manuellement le modèle au format `Provider#Model-name`
- Vérifiez que la clé API est valide
- Vérifiez que le client prend en charge l'endpoint de liste des modèles

### Erreurs d'authentification
- Vérifiez `Authorization: Bearer YOUR_API_KEY`
- Vérifiez l'endpoint `https://alterhq.com/api` ou `/api/v1`
- Vérifiez que votre compte est actif

### Problèmes de connexion
- Utilisez un client compatible OpenAI
- Vérifiez l'accès réseau à `alterhq.com`

### Latence élevée
- Essayez des modèles plus légers (`OpenAI#gpt-5-nano`, `Alter#light`)
- Utilisez le streaming pour les sorties longues

## Bonnes pratiques

1. Gardez les clés API dans des variables d'environnement (jamais en dur)
2. Surveillez le volume de requêtes et les taux d'erreur
3. Adaptez la capacité du modèle à la complexité de la tâche
4. Implémentez des retries avec backoff
5. Utilisez des modèles moins chers pour les tâches routinières
