<Info>
  La **fenêtre contextuelle** correspond à la quantité de texte qu'un modèle d'IA peut traiter au cours d'une seule conversation. Considérez-le comme la mémoire de travail du modèle : elle comprend vos messages, le contenu des fichiers, les définitions de Tools et les réponses de l'IA.
</Info>

## Pourquoi la fenêtre contextuelle est importante

Chaque interaction avec l'IA consomme de l'espace contextuel. Lorsque votre fenêtre contextuelle se remplit :

<CardGroup cols={3}>
  <Card title="Les performances ralentissent" icon="gauge">
    Les charges utiles plus volumineuses prennent plus de temps à traiter, ce qui augmente les temps de réponse
  </Card>
  <Card title="Le budget d’utilisation équitable s’épuise" icon="scale-balanced">
    Plus de jetons par demande consomment plus rapidement le budget de votre politique d'utilisation équitable
  </Card>
  <Card title="Le contexte est tronqué" icon="scissors">
    Les messages plus anciens peuvent être supprimés, ce qui amène l'IA à « oublier » les parties précédentes de la conversation.
  </Card>
</CardGroup>

Comprendre comment gérer votre fenêtre contextuelle vous aide à obtenir de meilleures performances et plus de valeur grâce à Alter.

---

## Ce qui consomme de l'espace contextuel

### 1. Historique des conversations

Chaque message que vous envoyez et chaque réponse que vous recevez reste dans son contexte. Les longues conversations consomment naturellement plus d’espace.

### 2. Contenu du fichier

Lorsque vous joignez des fichiers ou des dossiers, tout leur contenu est injecté dans le contexte :

- **Fichiers texte** — Le contenu complet est inclus
- **Fichiers de code** — L'intégralité du contenu du fichier est envoyé
- **Documents** — Le texte analysé est inclus
- **Images** — Des descriptions ou du texte OCR sont ajoutés

### 3. Définitions des Tools

Chaque Tool activé consomme de l'espace contextuel. Une définition de Tool comprend :

- Nom et description du Tool
- Paramètres obligatoires et facultatifs
- Descriptions et types de paramètres

<Warning>
  **Les Tools sont des consommateurs de contexte silencieux.** L'activation de plus de 10 Tools peut facilement consommer 30 à 50 % de votre fenêtre contextuelle avant même de commencer à taper.
</Warning>

### 4. Instructions système

Les invites en arrière-plan, les documents de base et les instructions système consomment tous de l'espace.

---

## Comment les Tools consomment le contexte

Voici un exemple concret de l'impact des Tools sur votre contexte :

| Tools activés | Contexte approximatif utilisé | Restant pour votre contenu |
|--------------|-------------------------|---------------------------|
| 0 Tools | ~5% | ~95% |
| 5 Tools | ~15% | ~85% |
| 10 Tools | ~30% | ~70% |
| 20 Tools | ~50% | ~50% |
| Plus de 30 Tools | ~70%+ | ~30% |

<Note>
  Ce sont des valeurs approximatives. Actual usage varies based on tool complexity and description length.
</Note>

## Stratégies de gestion du contexte

### 1. Utiliser Flow pour l'orchestration des Tools (recommandé)

Le moyen le plus efficace de réduire la consommation de contexte liée aux Tools consiste à utiliser **Flow** — l'orchestrateur de Tools intégré d'Alter.

<Card title="Charges de flux comme un seul Tool" icon="wand-magic-sparkles" href="/fr/guides/use-flow-orchestration">
  Au lieu de charger tous vos Tools activés dans chaque requête, Flow agit comme un Tool unique qui sélectionne et orchestre intelligemment les bons Tools pour chaque tâche.
  
  **Avantages:**
  - Réduit l'utilisation du contexte de 60 à 80 %
  - Améliore les temps de réponse
  - Gère automatiquement les tâches en plusieurs étapes
  - Permet d'économiser le budget d'utilisation équitable
</Card>

**Bonne pratique :** Configurez votre action par défaut « Demander n'importe quoi » pour utiliser uniquement Flow au lieu de tous les Tools activés.

### 2. Désactivez les Tools inutilisés

Activez uniquement les Tools que vous utilisez activement :

<Steps>
  <Step title="Revoyez vos Tools">
    Accédez à **Paramètres > Tools Manager** et voyez ce qui est activé.
  </Step>
  <Step title="Désactiver les Tools inutilisés">
    Désactivez les Tools que vous n'avez pas utilisés au cours de la semaine dernière
  </Step>
  <Step title="Réactiver si nécessaire">
    Vous pouvez toujours réactiver les Tools : ils se souviennent de votre authentification
  </Step>
</Steps>

### 3. Personnaliser les ensembles de Tools pour les actions

Chaque action peut être configurée avec ses propres Tools spécifiques. Il s'agit d'un moyen puissant de contrôler l'utilisation du contexte :

#### Demandez n'importe quoi

L'action **Demander n'importe quoi** est l'action utilisée par défaut lorsque vous tapez dans la zone d'invite sans sélectionner d'action spécifique. Il s'exécute partout dans Alter : réduisez votre sélection de Tools ici pour un impact maximal.

<Steps>
  <Step title="Ouvrir l'éditeur d'actions">
    Accédez à **Paramètres > Actions**.
  </Step>
  <Step title="Trouver, demander n'importe quoi">
    Sélectionnez l'action "Demander n'importe quoi"
  </Step>
  <Step title="Cliquez sur Avancé">
    Développez la section Avancé
  </Step>
  <Step title="Tools de limitation">
    Sélectionnez uniquement les Tools dont vous avez absolument besoin (ou simplement Flow)
  </Step>
</Steps>

#### Actions personnalisées

Lorsque vous créez des actions personnalisées, réfléchissez bien aux Tools réellement nécessaires :

<CardGroup cols={2}>
  <Card title="Apprendre des valeurs par défaut" icon="book-open">
    La plupart des actions Alter intégrées ont des Tools désactivés par défaut. Par exemple, « Corriger la grammaire » ne nécessite aucun Tool : il s'agit d'une pure transformation de texte.
  </Card>
  <Card title="Soyez minimal" icon="filter">
    Activez uniquement les Tools dont votre flux de travail spécifique a besoin. Si votre action formate simplement le texte, vous n'avez probablement besoin d'aucun Tool.
  </Card>
</CardGroup>

#### Flux vs Tools spécifiques

<Info>
  **Utilisez Flow** lorsque vous souhaitez de la flexibilité et que vous ne savez pas de quels Tools spécifiques vous aurez besoin. Il gère automatiquement la sélection des Tools.
</Info>

<Info>
  **Utilisez des Tools spécifiques** lorsque vous savez exactement de quels Tools votre flux de travail a besoin. Cela donne de meilleures performances et moins de va-et-vient avec l’IA.
</Info>

**Exemple :** Si vous créez une action qui crée uniquement des événements de calendrier, activez uniquement le Tool Calendrier plutôt que Flow. Cela élimine l’étape de sélection du Tool et s’exécute plus rapidement.

### 4. Utilisez les Workspaces pour les fichiers volumineux

Au lieu de joindre des fichiers volumineux à chaque conversation :

- Créer un **Workspace** pour les projets contenant des fichiers volumineux
- Référencer l'Workspace si nécessaire
- Effacer le contexte de l'Workspace lors du changement de tâche

<Card title="En savoir plus sur les Workspaces" icon="folder-open" href="/fr/guides/core-features#workspaces">
  Les Workspaces aident à organiser les fichiers et à gérer le contexte plus efficacement
</Card>

### 4. Effacer régulièrement le contexte

Utilisez **⌘ K** (Commande + K) pour effacer tous les contextes lorsque :

- Démarrer une nouvelle tâche sans rapport
- Le contexte précédent n'est plus pertinent
- Les réponses deviennent lentes

### 5. Résumer de longues conversations

Pour les très longues conversations :

- Exporter la conversation vers un fichier
- Démarrer une nouvelle conversation avec un résumé
- Référencez le fichier exporté si nécessaire

### 6. Soyez sélectif avec les pièces jointes

Au lieu de joindre des dossiers entiers :

- Joignez uniquement les fichiers spécifiques dont vous avez besoin
- Supprimer les fichiers du contexte lorsque vous en avez terminé avec eux

## Utilisation du contexte de surveillance

Bien qu'Alter n'affiche pas de compteur de contexte en temps réel, vous pouvez surveiller l'utilisation indirectement :

<CardGroup cols={2}>
  <Card title="Surveiller les temps de réponse" icon="clock">
    Des réponses plus lentes indiquent souvent une utilisation élevée du contexte
  </Card>
  <Card title="Vérifier le nombre de Tools" icon="toolbox">
    Plus de Tools activés = plus de contexte consommé. Consultez régulièrement le Tools Manager.
  </Card>
</CardGroup>

## Liste de contrôle des gains rapides

<Steps>
  <Step title="Activer le flux" icon="check">
    Configurez « Ask Anything » pour utiliser uniquement Flow au lieu de tous les Tools
  </Step>
  <Step title="Personnaliser les actions" icon="check">
    Définissez des Tools spécifiques pour les actions personnalisées – désactivez les Tools dont vous n'avez pas besoin
  </Step>
  <Step title="Tools d'audit" icon="check">
    Désactivez les Tools que vous n'avez pas utilisés récemment
  </Step>
  <Step title="Utiliser les Workspaces" icon="check">
    Organisez les grands projets dans des Workspaces au lieu de joindre des fichiers ad hoc
  </Step>
  <Step title="Contexte clair" icon="check">
    Appuyez sur **⌘ K** lorsque vous démarrez de nouvelles tâches
  </Step>
</Steps>
