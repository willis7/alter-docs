Por padrão, a Alter oferece uma variedade de modelos, listados alfabeticamente por sua empresa de hospedagem.

Você também pode usar sua [chave API](#via-your-api-key) ou usar [seu modelo local](#locally-via-ollama-or-studiolm).

### Escolher o seu modelo

Depois de abrir o Alter, digite `/` para mostrar os modelos disponíveis e selecione o modelo que deseja usar.


#### Escolhendo um modelo favorito

Selecione o ícone estrela ao lado de um modelo com o mouse para marcar o modelo como seu favorito.
![Gostaria de um modelo](/images/models/favorite-model.webp)


Ao usar sua chave de API ou um modelo local, estes aparecem no topo sob **Custom**.
Todas as outras interações são roteadas através da Nuvem Alter.

### Usando Alter Cloud

Por padrão, Alter é configurado com `/best` , nosso roteador de modelo interno, escolhendo o melhor modelo com base em sua consulta.

![Router e exemplo de modelo](/images/settings/router.png)


Você pode alterar o modelo padrão em **Settings > Defaults > Model**.

### Usando sua chave de API

![Personalizado Local](/images/models/custom-api-keys.webp)

Você pode usar sua própria chave de API para se conectar diretamente ao endpoint do seu provedor:
1. Vá para **Configurações > Chaves de API > Fornecedor personalizado**.
2. Ativar o Fornecedor Personalizado
3. Escolha seu provedor na lista

> Uma vez conectado, um tick aparecerá ao lado de **Endpoint personalizado**.

Os modelos do seu provedor serão listados na seção **Custom**.

> Só suportamos endpoints compatíveis com OpenAI, o que significa que você não pode usar suas chaves de API antrópicas.

Aqui está uma lista das plataformas suportadas para chaves personalizadas, com base na imagem fornecida:

1.  Azure
2.  Gemini
3.  Estúdio LM
4.  Mistral
5.  Ollama
6.  OpenAI
7.  Abrir o Roteador
8.  Personalizado (Qualquer endpoint compatível com OpenAI)

#### Configurar um ponto final do Azure no Alter

1.  ** Ativar provedor personalizado:** Nas configurações do Alter, habilite a opção "fornecedor personalizado".
2.  ** Escolha Azure:** Selecione Azure como seu provedor.
3.  **Enter Terminal completo do Chat URL do ponto final:** Forneça o URL completo do endpoint do chat, incluindo o parâmetro `api-version` e o nome do modelo dentro do URL. Por exemplo: medicine
4.  **Enter API Key:** Digite sua chave Azure OpenAI API.

Considerações Importantes

*   **Exigência de Endpoint completo:** Certifique-se de usar o endpoint de finalização do chat *complete*, pois Alter extrai o nome do modelo da URL.
*   ** Nome do modelo em URL:** O nome de implantação do modelo * deve * ser incluído no caminho URL.
*   ** Versão API:** A URL * deve * incluir o parâmetro `api-version`.

### Localmente usando Ollama ou LM Studio

![Personalizado Local](https://beehiiv-images-production.s3.amazonaws.com/uploads/asset/file/a3cb944c-7379-49a6-9349-c829bcda071b/image.png?t=1740499713)

Para usar Alter com modelos locais (via Ollama ou LM Studio):
1. Vá para **Configurações > Chaves de API > Fornecedor personalizado**.
2. Ativar o Fornecedor Personalizado
3. Escolha Ollama ou LM Studio na lista de provedores

> Uma vez conectado, um tick aparecerá ao lado de **Endpoint personalizado**.
