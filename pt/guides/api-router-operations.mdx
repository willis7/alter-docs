## Limites de uso

Em fair use:
- Limite diario: `200` requisicoes/dia
- Apos o limite: throttling para `1 requisicao / 10 minutos`
- Para volume estavel mais alto: faca top-up de budget

<Note>
  O API Router foi feito para uso pessoal e cargas de baixo volume. Para produtos multiusuario ou trafego agentic pesado, use infraestrutura dedicada do provider.
</Note>

## Problemas comuns

### Modelos nao listados
- Defina o modelo manualmente no formato `Provider#Model-name`
- Verifique se a API key e valida
- Verifique se o cliente suporta endpoint de listagem de modelos

### Erros de autenticacao
- Confirme `Authorization: Bearer YOUR_API_KEY`
- Confirme o endpoint `https://alterhq.com/api` ou `/api/v1`
- Confirme que sua conta esta ativa

### Problemas de conexao
- Use um cliente compativel com OpenAI
- Verifique acesso de rede a `alterhq.com`

### Alta latencia
- Tente modelos mais leves (`OpenAI#gpt-5-nano`, `Alter#light`)
- Use streaming para saidas longas

## Boas praticas

1. Guarde API keys em variaveis de ambiente (nunca hardcode)
2. Monitore volume de requisicoes e taxas de erro
3. Combine capacidade do modelo com complexidade da tarefa
4. Implemente retries com backoff
5. Use modelos de menor custo para tarefas rotineiras
