## Nutzungslimits

Unter fair use:
- Tageslimit: `200` Anfragen/Tag
- Nach dem Limit: gedrosselt auf `1 Anfrage / 10 Minuten`
- Für dauerhaft höheres Volumen: Budget aufladen

<Note>
  API Router ist für persönliche Nutzung und niedriges Volumen gedacht. Für Multi-User-Produkte oder hohen agentic Traffic sollten Sie dedizierte Provider-Infrastruktur verwenden.
</Note>

## Häufige Probleme

### Modelle nicht gelistet
- Modell manuell im Format `Provider#Model-name` setzen
- Prüfen, ob der API-Schlüssel gültig ist
- Prüfen, ob der Client Modell-Listing-Endpoints unterstützt

### Authentifizierungsfehler
- `Authorization: Bearer YOUR_API_KEY` prüfen
- Endpoint `https://alterhq.com/api` oder `/api/v1` prüfen
- Prüfen, ob Ihr Konto aktiv ist

### Verbindungsprobleme
- OpenAI-kompatiblen Client verwenden
- Netzwerkzugriff auf `alterhq.com` prüfen

### Hohe Latenz
- Leichtere Modelle testen (`OpenAI#gpt-5-nano`, `Alter#light`)
- Für lange Antworten Streaming verwenden

## Best Practices

1. API-Schlüssel in Umgebungsvariablen speichern (nie hardcoden)
2. Anfragevolumen und Fehlerraten überwachen
3. Modellfähigkeit an Aufgabenkomplexität anpassen
4. Retries mit Backoff implementieren
5. Für Routineaufgaben günstigere Modelle nutzen
